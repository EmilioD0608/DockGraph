## ğŸ› Bug Reports

We use GitHub issues to track public bugs. If you report a bug, please help us by providing as much of the following information as possible:

1.  **A clear and descriptive title.**
2.  **Steps to reproduce the issue.** (e.g., "1. Click on 'X', 2. Enter 'Y'...")
3.  **Expected behavior.**
4.  **Actual behavior.**
5.  **Screenshots or recordings** if relevant.
6.  **Environment details:**
    *   OS (Windows, macOS, Linux)
    *   Node.js version
    *   Browser (Chrome, Firefox, etc.)

## ğŸ› ï¸ Development Workflow

1.  **Fork the repository** to your own GitHub account.
2.  **Clone the project** to your local machine.
3.  **Create a branch** for your fix or feature: `git checkout -b feature/amazing-feature`.
4.  **Install dependencies** as described in the [README](README.md).
5.  **Make your changes.**
6.  **Test your changes.** Ensure the application still runs and passes any tests.
7.  **Commit your changes** with a descriptive commit message.
8.  **Push to your branch**: `git push origin feature/amazing-feature`.
9.  **Open a Pull Request** on the main repository.
